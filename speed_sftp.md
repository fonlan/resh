# SFTP 速度优化实现计划（OpenSSH 兼容）

## 1. 目标与边界

### 目标
- 在不改服务端（兼容普通 Linux 自带 OpenSSH `sftp-server`）前提下，提升上传/下载吞吐与稳定性。
- 优先提升高 RTT 链路与大文件传输表现，同时兼顾小文件批量场景。
- 保持现有完整性保障（短读检测、EOF 前不足、远端文件尺寸漂移检测、取消与错误可见）。

### 非目标
- 不依赖服务端私有补丁。
- 不做“默认激进”的多连接分片下载。
- 不破坏当前 UI/任务模型与进度事件协议。

## 2. 现状摘要（基于当前代码）
- 下载路径：单请求串行（`256KB`），属于单 flight 模式。
- 上传路径：`32KB` 分块，最多 `16` 个并发写请求。
- SFTP 请求超时：`60s`。
- 已有历史结论：过于激进的下载并发在部分服务器/链路上会出现首包后长时间卡顿，因此必须具备保守回退策略。

## 3. 总体策略（推荐）
- 先做单连接内请求流水线（same handle，多 in-flight request），再做多连接并行。
- 先做能力探测驱动（`limits@openssh.com` + 本地上限），再做参数调优。
- 先做“默认稳态参数 + 自适应降级”，再开放高级开关。

## 4. 分阶段里程碑

## M0：基线与观测（必须先做）
- 在下载/上传路径补充结构化诊断埋点：请求大小、in-flight、超时次数、重试次数、平均 RTT 估计、有效吞吐。
- 统一速度口径：显示速度与调度决策使用同一采样窗口，避免“显示快/调度慢”或相反。
- 建立实验矩阵脚本（本地可复现）：低延迟/高延迟、大文件/小文件、低丢包/高抖动。

### 交付物
- 一份基线报告（当前默认参数在各场景下的吞吐、耗时、超时率）。
- 调试日志字段说明文档（方便后续定位回归）。

### 验收
- 能稳定复现当前瓶颈，并可量化对比后续阶段收益。

## M1：能力探测与参数模型
- 在会话建立后读取并缓存服务端能力：`max-packet-length`、`max-read-length`、`max-write-length`、`max-open-handles`。
- 设计“本地策略上限 + 服务端上限”的最终计算规则。
- 引入传输参数 profile（`safe` / `balanced` / `fast`），默认 `balanced`，并允许自动降级到 `safe`。

### 交付物
- 传输参数计算器（运行时得出 chunk/in-flight 上限）。
- 会话级能力缓存与失效机制（断线重连后重算）。

### 验收
- 在不同 OpenSSH 版本/发行版下均能正常运行，且不会发送超过服务端上限的请求。

## M2：下载提速（首个核心收益）
- 将下载从单 flight 升级为“单连接、同一 handle 的有序流水线”：
- 发送端允许多个 `READ` 并发 in-flight（初始保守值，例如 `4~8`）。
- 接收端按 offset 回写本地文件，确保最终文件有序一致。
- 引入自适应策略：
- 出现超时/异常时快速降并发；
- 连续稳定后缓慢升并发；
- 达到阈值后锁定稳态窗口。
- 保留完整性检查与取消语义不变。

### 交付物
- 新下载调度器（可切换 old/new 逻辑，便于灰度）。
- 兼容性回退开关（会话内自动回退并持久到会话生命周期）。

### 验收
- 高 RTT 大文件场景下载吞吐显著提升。
- 故障链路下失败可控，不出现“无输出长卡死”。

## M3：上传优化（稳定优先）
- 上传分块从固定 `32KB` 改为“能力探测驱动的动态块大小”。
- 并发请求窗口支持自适应调节，但上限保守，避免写入超时风暴。
- 维持现有写超时机制，同时增加“连续超时降级”策略。

### 交付物
- 上传参数调度器（与下载共享部分策略模块）。
- 上传稳定性回归报告（大文件、慢盘、高 RTT）。

### 验收
- 上传吞吐提升且超时率不高于基线。
- 传输完成后远端文件一致性检查通过。

## M4：多文件任务调度与连接级并行
- 场景定位为“海量小文件”；单文件默认仍优先单连接流水线。
- 在任务队列层引入受控多连接并行（如 `2~4` 连接），按会话资源与服务器能力限流。
- 避免连接风暴：同一服务器并发连接数受全局配置限制。

### 交付物
- 小文件批量模式策略（目录上传/下载启用）。
- 连接池与限流器（每服务器与全局双重配额）。

### 验收
- 小文件批量场景总耗时明显下降。
- 不触发服务器侧限流/封禁风险。

## M5：可选的大文件多连接分段（默认关闭）
- 仅在满足阈值条件时启用：文件足够大、链路稳定、用户或策略允许。
- 失败自动退回 M2 单连接流水线，不影响任务最终完成。
- UI 明确标注“实验性/高级模式”，默认不打扰普通用户。

### 交付物
- 分段策略开关与阈值配置。
- 自动回退路径与告警提示。

### 验收
- 在目标网络环境下有实测收益。
- 回退路径稳定，不引入不可恢复状态。

## 5. 配置与数据结构改造建议
- 新增 SFTP 速度策略配置：
- `transferProfile`: `safe | balanced | fast`
- `downloadMaxInflight`
- `uploadMaxInflight`
- `chunkSizeMin` / `chunkSizeMax`
- `enableMultiConnectionForSmallFiles`
- `enableLargeFileStriping`（默认 `false`）
- 会话运行态新增：
- 服务端能力缓存（limits）
- 自适应窗口状态（成功/超时统计、当前窗口）
- 回退状态（当前会话是否降级到 safe）

## 6. 兼容性与安全策略
- 所有自动调优都必须尊重服务端 `limits`。
- 默认策略必须对 OpenSSH 常见发行版稳定。
- 任何超时风暴必须快速降级，不允许无限重试。
- 错误信息必须用户可见且可定位（保留任务 ID、偏移、阶段）。

## 7. 测试与验收矩阵
- 文件类型：
- 单大文件（1GB 级）。
- 中等文件（100MB 级）。
- 海量小文件（1KB~256KB，多目录）。
- 网络环境：
- 低 RTT（<5ms）。
- 中 RTT（30~80ms）。
- 高 RTT（120~250ms）。
- 干扰环境：
- 抖动、限速、轻微丢包。
- 验收指标：
- 吞吐提升比例。
- 端到端耗时。
- 超时率/重试率。
- 失败可恢复率。
- 文件完整性校验通过率。

## 9. 风险清单与缓解
- 风险：高并发导致服务器响应抖动或超时。
- 缓解：保守初始窗口 + 连续失败快速降级 + 会话级记忆回退。
- 风险：顺序写入与乱序数据处理复杂度提高。
- 缓解：严格 offset 校验、集中写入协调器、失败即中止并清晰报错。
- 风险：多连接策略触发服务器连接限制。
- 缓解：全局与主机双重配额，默认低并发连接数。
